# React & Next.js CVE-2025-55182: 対策とセキュリティガイド

React Server Components (RSC) の利用が拡大する中、Meta の Bug Bounty プログラムを通じて発見された CVE-2025-55182 は、開発者にとって重要なセキュリティリスクとなります。本記事では、この脆弱性とその影響、そして迅速な対応と対策について解説します。

## 脆弱性の概要

CVE-2025-55182 は、React Server Components (RSC) を利用する際に、検証が不十分なリクエストを受け付けることで、Remote Code Execution (RCE) 脆弱性につながる可能性があります。特に、Server Function エンドポイントを実装していなくても、RSC をサポートしている場合、この脆弱性が悪用されるリスクがあります。

Vercel のセキュリティアドバイザリにも、攻撃者が細工したリクエストを送信することで、検証が不十分なまま RCE に繋がる可能性があると指摘されています。これは、RSC がクライアントサイドのデータに対して過度な信頼を置いている場合に発生するリスクです。

## 影響を受けるバージョン

この脆弱性は、以下のバージョンで使用している React および Next.js プロジェクトに影響を及ぼします。

#### React関連パッケージ

| パッケージ名 | 影響を受けるバージョン |
| :--- | :--- |
| `react-server-dom-webpack` | 19.0.0, 19.1.0, 19.1.1, 19.2.0 |
| `react-server-dom-parcel` | 同上 |
| `react-server-dom-turbopack` | 同上 |

#### Next.jsの影響を受けるバージョン

* **Next.js 15.x**
* **Next.js 16.x**
* **14.3.0-canary.77 以降** の canary 版

特に、`create-next-app` で作成した Next.js 15.x/16.x アプリケーションは、App Router がデフォルトで有効になっているため、**デフォルト設定のまま脆弱です**。App Router の利用を検討している場合は、この脆弱性を念頭に置く必要があります。

今回の脆弱性は、報告から公開まで迅速に対応されました。

| 日付 | 出来事 |

## 対策と具体的な対応

この脆弱性への対策として、以下の手順を実施することが推奨されます。

1.  **バージョンアップ:** 影響を受ける React および Next.js パッケージのバージョンを、修正バージョンにアップデートしてください。
2.  **App Router の設定見直し:** Next.js 15.x/16.x で App Router を利用している場合は、デフォルト設定を維持せず、必要に応じて設定を見直してください。特に、クライアントサイドからのデータに対する検証を強化することを検討してください。
3.  **入力値の検証:**  クライアントサイドからのすべての入力値に対して、厳格な検証を行うようにしてください。入力値の型、長さ、パターンなどを検証し、悪意のあるデータが処理されないようにします。
4.  **Content Security Policy (CSP) の設定:**  CSP を設定し、許可されたソースからのリソースのみを読み込めるようにすることで、XSS 攻撃のリスクを軽減できます。
5.  **セキュリティスキャナの導入:**  定期的にアプリケーションをセキュリティスキャンし、潜在的な脆弱性を早期に発見できるようにします。
6.  **サーバーサイドでのデータ処理:** 可能な限り、クライアントサイドのデータではなく、サーバーサイドでデータ処理を行うようにすることで、クライアントサイドの脆弱性を回避できます。

## まとめ

CVE-2025-55182 は、React Server Components を利用する上で無視できないセキュリティリスクです。  この脆弱性への迅速な対応と、継続的なセキュリティ対策の実施が重要となります。 特に、App Router の設定、入力値の検証、セキュリティスキャナの導入などを徹底することで、安全なアプリケーション開発を実現できます。  最新のセキュリティ情報を常に把握し、適切な対策を講じることで、セキュリティリスクを最小限に抑えることができます。
